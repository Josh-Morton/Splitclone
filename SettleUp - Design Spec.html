Design principles

Five rules that keep the app calm, fast to build, and effortless to maintain by one person.

1 · Calm & money-first

Lots of white space, one clear number per screen. The balance is always the hero. Money never feels stressful.

2 · Mobile-first, thumb-friendly

Designed for one hand on a phone. Primary actions sit low; tap targets ≥ 44px. A bottom tab bar and a single floating “add” button.

3 · One primary colour

A single blue for all actions; green/red only for owed/owing. No rainbow. Easy to keep consistent.

4 · Tailwind defaults only

Tokens map to stock Tailwind classes and one tiny theme extension — no custom CSS framework to maintain.

5 · Components, not pages

A small kit (button, input, card, list-row, chip, sheet) composes every screen, so the UI stays consistent for free.

Build & maintain it in Tailwind

So this spec is trivial to keep in sync with the code: every token is one line in tailwind.config.js, and every component is a thin wrapper. No design system to maintain — just these.

tailwind.config.js — theme.extend.colors
  • brand: '#1F3A5F' · primary: '#2E75B6' · surface: '#EAF2FB'
  • owed: '#2E7D4F' · owe: '#B23A3A' · bg: '#F4F6FA'
  • Everything else uses stock Tailwind slate-*. Radius: rounded-2xl cards, rounded-xl controls.

Recommended libraries (all free, low-maintenance):
  • Tailwind CSS for all styling — no separate CSS files.
  • lucide-react for icons (the emoji here are placeholders for clean line icons).
  • Headless UI or Radix only for the bottom-sheet/modal and toggles — accessible, unstyled, you keep full control.
  • Skip a heavy component library; this small kit + Tailwind is faster to maintain solo.
Layout rules to keep it consistent:
  • One screen = one route. Persistent bottom tab bar: Home · Expenses · List · Reports. Single FAB for “add expense”.
  • 16px screen padding, 12px gap between cards, 44px minimum tap target.
  • Add/Settle forms open as bottom sheets, not new pages — faster and more native-feeling.
  • Sync status always visible on Home (a small pill) so offline state is never a surprise.
